Authentication

All API requests require an API key passed via the X-API-Key header, or a JWT Bearer token in the Authorization header.

Getting started: Register at POST /api/v1/register with email + password, then create an API key at POST /api/v1/keys.

Quick Start

# Register
curl -X POST https://your-host/api/v1/register \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"s3cure!"}'

# Create API key
curl -X POST https://your-host/api/v1/keys \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{"label":"my-app","tier":"free"}'

# Upscale an image
curl -X POST https://your-host/api/v1/upscale \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"image_data":"BASE64_IMAGE","model":"natural","format":"png"}'

Webhooks

MethodPathDescription
POST/api/v1/webhooksRegister a webhook endpoint
GET/api/v1/webhooksList your registered webhooks
DELETE/api/v1/webhooks/:idRemove a webhook
Webhook payloads are signed with HMAC-SHA256 via the X-SRGAN-Signature header.
Events: job.completed, job.failed, batch.completed. Retries: 3 attempts with exponential backoff.

Waifu2x Options

When using any waifu2x model, you can pass these additional fields in the upscale request:

FieldTypeDefaultDescription
waifu2x_noise_level0–31Noise reduction strength (0=none, 1=light, 2=medium, 3=aggressive)
waifu2x_scale1 or 221=denoise only, 2=upscale 2×
waifu2x_stylestring"anime""anime", "photo", or "artwork" — selects weight set / sharpening profile
// Waifu2x example: denoise + upscale 2× with photo style
{
  "image_data": "BASE64...",
  "model": "waifu2x",
  "waifu2x_noise_level": 2,
  "waifu2x_scale": 2,
  "waifu2x_style": "photo"
}
